# Руководство по тестированию: Квиз «Первая»

## О проекте

Квиз-лендинг школы тенниса «Первая». Один файл `index.html` (HTML + CSS + JS inline). Пользователь проходит 4 вопроса, видит персонализированный результат, оставляет контакты, переходит в Telegram.

**Стек:** Vanilla HTML/CSS/JS, без фреймворков, без сборки.
**Деплой:** Vercel (конфиг `vercel.json`).
**Трафик:** 85–95% мобильный, дизайн mobile-first.

---

## Как запустить

1. Клонировать репозиторий
2. Открыть `index.html` в браузере (двойной клик или через Live Server в VS Code)
3. Никакой установки зависимостей не требуется — всё в одном файле

Для проверки деплоя:
```bash
npm i -g vercel
vercel dev
```

### TC-02: Навигация вперёд — полный проход

**Шаги:**

1. Клик «Пройти тест бесплатно» → **Ожидание:** показан `screen-q1`, навбар «Шаг 1 из 4», прогресс 25%
2. Клик по любой карточке в Q1 → карточка подсвечивается (класс `selected`), через ~300 мс переход на `screen-q2`, навбар «Шаг 2 из 4», прогресс 50%
3. Клик по любой карточке в Q2 → через ~300 мс переход на `screen-q3`, навбар «Шаг 3 из 4», прогресс 75%
4. Клик по любой карточке в Q3 → через ~300 мс переход на `screen-q4`, навбар «Шаг 4 из 4», прогресс 100%
5. Клик по любой карточке в Q4 → через ~300 мс переход на `screen-loading`
6. Экран загрузки: 3 этапа появляются последовательно (галочки ✓), прогресс-бар заполняется → через ~2.8 сек автопереход на `screen-result`
7. Экран результата: заголовок и описание персонализированы, видны 2 подарка и форма
8. Заполнить имя и телефон, нажать «Получить разбор + 2 подарка» → через ~500 мс переход на `screen-final`
9. Финальный экран: конфетти, кнопка «Забрать подарки в Telegram»

### TC-10: Форма — валидация

**Поле «Имя» (`#inName`):**

| Ввод | Ожидание |
|------|----------|
| Пусто | Красная рамка (`.error`), текст «Введите имя» (`.visible`) |
| 1 символ (например «А») | Ошибка — `name.length < 2` |
| 2+ символа (например «Ал») | Валидация пройдена |

**Поле «Телефон» (`#inPhone`):**

| Ввод | Ожидание |
|------|----------|
| Пусто | Красная рамка, текст «Введите корректный номер» |
| Менее 11 цифр | Ошибка — `digits.length < 11` |
| 11 цифр (полный номер) | Валидация пройдена |

**Очистка ошибок:** При начале ввода в поле с ошибкой — красная рамка и текст ошибки исчезают.

### TC-11: Маска телефона

Маска реализована на JS (обработчики `input`, `focus`, `keydown` на `#inPhone`).

| Действие | Ожидание |
|----------|----------|
| Фокус на пустое поле | Появляется «+7» |
| Ввод «9161234567» | Форматируется: «+7 (916) 123-45-67» |
| Ввод начиная с «8» | «8» заменяется на «7»: ввод «89161234567» → «+7 (916) 123-45-67» |
| Ввод начиная с цифры ≠ 7 и ≠ 8 | Автоматически добавляется «7» впереди |
| Backspace при значении «+7» или «+7 » | Поле очищается полностью |
| Ввод нецифровых символов | Игнорируются (`.replace(/\D/g, '')`) |

### TC-14: CSS — адаптив и брейкпоинты

Три брейкпоинта в CSS:

| Условие | Что меняется |
|---------|-------------|
| `min-width: 640px` | `h1` → 40px, `h2` вопросов → 30px |
| `max-width: 480px` | Отзывы → 1 колонка, карточки ответов → 1 колонка, `h1` → 27px |
| По умолчанию (481–639px) | Отзывы → 2 колонки, карточки → 2 колонки, `h1` → 32px |

**Устройства для проверки:**
- iPhone SE (375×667) — мобильная раскладка (<480px)
- iPhone 12/13 (390×844)
- iPad (768×1024) — десктопная раскладка (>640px)
