# Проект: Квиз для школы тенниса "Первая"

## Описание
Квиз-лендинг (одностраничник). Пользователь проходит 4 вопроса → видит персонализированный результат → оставляет контакты → переходит в Telegram, где подключается нейропродавец (бот).

## Воронка
Реклама (VK/Telegram/Директ) → Квиз (index.html) → Telegram (личный аккаунт + бот-нейропродавец) → Пробное занятие → Оплата

## Файлы проекта

| Файл | Назначение |
|------|-----------|

| `brief.md` | Пошаговый план разработки: экраны, элементы, переходы, стиль, чеклист |
| `research.md` | Исследование: 10+ примеров квизов, UX-анализ, экспертная оценка, кейсы с метриками |

| `конкурентный-анализ-москва.md` | Конкурентный анализ (Москва) |

| `.vercelignore` | Файлы, исключённые из деплоя |

## Структура index.html

### Экраны (навигация)

```
screen-start    → Стартовая страница (CTA → screen-q1)
screen-q1       → Вопрос 1: Для кого подбираем? (сегмент)
screen-q2       → Вопрос 2: Что развивает теннис? (образовательный)
screen-q3       → Вопрос 3: Что останавливает? (барьер, ветвление по Q1)
screen-q4       → Вопрос 4: Где удобнее? (локация)
screen-loading  → Экран загрузки "Подбираем программу..." (2.5 сек)
screen-result   → Результат + форма контактов (персонализация по Q1)
screen-final    → Финальный экран → переход в Telegram
```

### Переходы между экранами

```
start ──[CTA]──→ q1
q1 ──[карточка]──→ q2       q1 ──[Назад]──→ start
q2 ──[карточка]──→ q3       q2 ──[Назад]──→ q1
q3 ──[карточка]──→ q4       q3 ──[Назад]──→ q2
q4 ──[карточка]──→ loading   q4 ──[Назад]──→ q3
loading ──[авто]──→ result
result ──[форма]──→ final
final ──[CTA]──→ Telegram (внешняя ссылка)
```

### Структура CSS (внутри `<style>`)

```
:root             — CSS-переменные (цвета, радиусы)
body, h1-h3       — Базовые стили
.screen           — Переключение экранов (display: none/flex)
@keyframes        — Анимации (fadeIn, confettiFall)
.start-*          — Стартовый экран (badge, заголовок, stats, отзывы, CTA)
.q-nav, .q-*      — Навбар вопросов (прогресс-бар, кнопка "Назад")
.q-content        — Контент вопроса (заголовок, подсказка)
.answers-grid     — Сетка карточек ответов
.a-card           — Карточка ответа (обычная, hover, selected)
.result-*         — Экран результата (badge, заголовок, подарки)
.gift             — Карточка подарка
.form-area, .f-*  — Форма (инпуты, ошибки, кнопка)
.final-*          — Финальный экран
.tg-btn           — Кнопка Telegram
.confetti-piece   — Конфетти
@media            — Адаптив (640px+, 480px-)
```

### Структура JS (внутри `<script>`)

```
ans               — Объект с ответами пользователя {1: 'child', 2: 'coordination', ...}
TG                — Ссылка на Telegram (t.me/Roman_Lekomtsev)
show(id)          — Переключение экранов
startQuiz()       — Начало квиза (CTA на стартовой)
goBack(from)      — Навигация назад
pick(card, q)     — Выбор ответа + автопереход через 300мс
ph (input mask)   — Маска ввода телефона (+7)
send(e)           — Отправка формы, формирование Telegram-ссылки
boom()            — Анимация конфетти на финальном экране
```

## Где менять данные

| Что изменить | Где искать в index.html |
|-------------|----------------------|
| Telegram-ссылку | JS: `const TG = 'https://t.me/...'` |
| Тексты вопросов | HTML: `<h2>` внутри `#screen-q1..q4` |
| Варианты ответов | HTML: `.a-card` внутри `.answers-grid` |
| Тексты подарков | HTML: `.gift` внутри `#screen-result` |
| Цвета (палитра) | CSS: `:root { --accent, --bg, ... }` |
| Шрифты | HTML: `<link>` Google Fonts + CSS: `font-family` |
| Тексты стартовой | HTML: `#screen-start .start-inner` |
| Лейблы ответов для Telegram | JS: объект `L` внутри `send()` |
| Мета-теги (SEO, OG) | HTML: `<head>` — title, description, og:* |

## Деплой
- Платформа: Vercel
- Конфиг: `vercel.json`
- Деплоится только `index.html` (остальные .md файлы в `.vercelignore`)

## Стиль кода
- Весь проект — один файл `index.html` (HTML + CSS + JS inline)
- Комментарии в коде — на русском
- Vanilla JS, без фреймворков
- Mobile-first (85-95% трафика — мобильный)
